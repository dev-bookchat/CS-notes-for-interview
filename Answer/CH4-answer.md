## 1. 데이터베이스의 기본 🍑

### Q1. 데이터베이스와 DBMS에 대해 설명해보세요.
데이터베이스는 일정한 규칙, 혹은 규약을 통해 구조화되어 저장되는 데이터의 모음입니다. <br/>
해당 데이터베이스를 제어, 관리하는 통합 시스템을 DBMS라고 하며, 데이터베이스 안에 있는 데이터들은 특정 DBMS마다 <br/>
정의된 쿼리 언어를 통해 삽입,삭제,수정,조회 등을 수행할 수 있씁니다. <br/>
또한 데이터베이스는 실시간 접근과 동시 공유가 가능합니다. <br/>

### Q2. 릴레이션은 무엇입니까?
릴레이션은 데이터베이스에서 정보를 구분하여 저장하는 기본 단위입니다. <br/>
엔티티에 관한 데이터를 데이터베이스는 릴레이션 하나에 담아서 관리합니다.<br/>
릴레이션은 관계형 데이터베이스에서는 '테이블'이라 하며 NoSQL 데이터베이스에서는 '컬렉션'이라고 합니다.<br/>

### Q3. 속성과 도메인에 대해 설명해보세요.
속성(attribute)은 릴레이션에서 관리하는 구체적이며 고유한 이름을 갖는 정보입니다.<br/>
예를 들어'차'라는 엔터티의 속성을 뽑아보면 차 넘버, 바퀴 수, 차 색깔, 차종 등이 있습니다. <br/>
이 중에서 서비스의 요구 사항을 기반으로 관리해야하는 속성들만 엔티티의 속성이 됩니다. <br/>

도메인(domain)은 릴레이션에 포함된 각각의 속성들이 가질 수 있는 값의 집합을 말합니다. <br/>
예를 들어 과일이라는 속성이 있다면 이 속성이 가질 수 있는 값은 {사과,딸기}라는 집합이 될 수 있습니다.<br/>

### Q4. 테이블 안에서 필드와 레코드가 무엇인지 설명해보세요.
앞서 설명한 것(릴레이션,속성,도메인)등을 기반으로 데이터베이스에서 필드와 레코드로 구성된 테이블을 만들 수 있습니다.<br/>
멤버라는 테이블이 있을 때 회원이란 엔터티는 <br/>
이름, 나이, 전화번호 등의 속성을 가지며 이것은 필드로 분류됩니다. <br/>
그리고 필드에 쌓이는 각 정보들은 행(Row)단위의 데이터로 '레코드'라고 부릅니다.<br/>
레코드는 튜플이라고도 합니다. <br/>
필드는 데이터의 종류에 따라 타입을 갖습니다.각 타입은 각각의 최대 용량이 정해져 있습니다.<br/>

### Q5. TEXT와 BLOB의 차이를 설명해보세요.
두 개의 타입 모두 큰 데이터를 저장할 때 쓰는 타입으로 <br/>
TEXT는 주로 큰 문자열 저장에 쓰이며 게시판 본문 저장 등에 쓰입니다.<br/>
BLOB은 이미지,동영상 등 큰 데이터 저장에 쓰입니다. <br/>
그러나 보통은 아마존의 이미지 호스팅 서비스인 S3을 이용하는 등 서버에 파일을 올리고 <br/>
파일 경로를 VARCHAR(문자열 타입)으로 저장합니다.<br/>

### Q6. 테이블에서 '키'의 기능을 설명하고 키의 종류를 생각나는대로 말해보세요.
키는 테이블 간의 관계를 조금 더 명확하게 하고 테이블 자체의 인덱스를 위해 설정된 장치입니다.<br/>
기본키, 외래키, 후보키, 슈퍼키, 대체키가 있습니다.<br/>

## 2. ERD와 정규화 과정 🍑

### Q1. ERD는 무엇입니까? ERD의 장점과 단점도 설명해보세요.
ERD는 데이터베이스를 구축할 때 가장 기초적인 뼈대 역할을 하며, 릴레이션 간의 관계들을 정의한 것입니다.<br/>
만약 서비스를 구축한다면 가장 먼저 신경써야 합니다. <br/>
ERD는 시스템 요구사항을 기반으로 작성되며, 이 ERD를 기반으로 데이터베이스를 구축합니다. <br/>
데이터베이스를 구축한 후에도 디버깅 또는 비즈니스 프로세스 재설계가 필요한 경우 설계도 역할을 담당하기도 합니다. <br/>
그러므로 ERD는 관계형 구조로 표현할 수 있는 데이터를 구성하는 데에는 유용하다는 장점이 있지만 <br/>
비정형 데이터는 충분히 표현할 수 없는 단점도 있습니다. <br/>

### Q2. 정규화 과정에 대해 설명하고 종류를 생각나는대로 말해보세요.
정규화 과정은 릴레이션 간의 잘못된 종속 관계로 인해 데이터베이스 이상 현상이 일어나서 이를 해결하거나, 저장 공간을 효율적으로 사용하기 위해 <br/>
릴레이션을 여러개로 분리하는 과정입니다. <br/>
정규화 과정은 정규형 원칙을 기반으로 정규형을 만들어가는 과정이며, 정규화된 정도는 정규형으로 표현합니다.<br/>
기본 정규형인 제1정규형, 제2정규형, 제3정규형, 보이스/코드 정규형이 있고 고급 정규형인 제4정규형, 제5정규형이 있습니다.<br/>

### Q3. 정규형의 원칙을 설명해보세요.
정규형의 원칙이란 같은 의미를 표현하는 릴레이션이지만 좀 더 좋은 구조로 만들어야 하고,<br/>
자료의 중복성은 감소해야 하고, 독립적인 관계는 별개의 릴레이션으로 표현해야 하며,<br/>
각각의 릴레이션은 독립적인 표현이 가능해야 하는 것을 말합니다.<br/>

### Q4. 제3정규형에 대해 설명해보세요.
제2정규형이고 기본키가 아닌 모든 속성이 이행적 함수 종속을 만족하지 않는 상태를 말합니다.<br/>
이행적 함수 종속이란 A -> B와 B -> C 가 존재하면 논리적으로 A -> C가 성립하는데 <br/>
이 때 집합 C가 집합 A에 이행적으로 함수 종속이 되었다고 합니다. <br/>

### Q5. 보이스/코드 정규형에 대해 설명해보세요.
보이스/코드 정규형(BCNF)는 제3정규형이고, <br/>
결정자가 후보키가 아닌 함수 종속 관계를 제거하여 <br/>
릴레이션의 함수 종속 관계에서 <br/>
모든 결정자가 후보키인 상태를 말합니다.<br/>

## 3. 트랜잭션과 무결성 🍑

### Q1. 트랜잭션에 대해 설명하고, ACID 특징의 종류를 생각나는대로 말해보세요.
트랜젝션은 데이터베이스에서 하나의 논리적 기능을 수행하기 위한 작업의 단위를 말하며<br/>
데이터베이스에 접근하는 방법은 쿼리이므로, 즉 여러 개의 쿼리들을 하나로 묶는 단위를 말합니다. <br/>
이에 대한 특징은 원자성, 일관성, 독립성, 지속성이 있으며 이를 한꺼번에 ACID 특징이라고 합니다.<br/>

### Q2. 원자성에 대해 설명해 보세요.
원자성은 트랜잭션과 관련된 일이 모두 수행되었거나 되지 않았거나를 보장하는 특징입니다. <br/>
예를 들어 트랜잭션을 커밋했는데, 문제가 발생하여 롤백하는 경우 그 이후에 모두 수행되지 않음을 보장하는 것을 뜻합니다.<br/>

### Q3. 트랜잭션 전파는 무엇입니까?
트랜잭션을 수행할 때 커넥션 단위로 수행하기 때문에 커넥션 객체를 넘겨서 수행해야 합니다.<br/>
하지만 이를 매번 넘겨주기가 어렵거나 귀찮으므로, 커넥션을 넘기지 않고 <br/>
여러 트랜잭션 관련 메서드의 호출을 하나의 트랜젝션에 묶이도록 하는 것을 트랜잭션 전파라고 합니다. <br/>

### Q4. 격리성에 대해 얘기해보고 왜 트랜잭션을 격리를 해야하는지 설명해보세요.
격리성은 트랜잭션 수행 시 서로 끼어들지 못하는 것을 말합니다. <br/>
복수의 병렬 트랜잭션을 실행시킬 때 순서대로가 아닌 성능이 좋은 순서대로 실행을 하지만<br/>
마치 순차적으로 실행되는 것처럼 작동시키기 위해 트랜잭션을 격리합니다.<br/>
그러기 우해 격리성은 트랜 잭션을 여러 개의 격리 수준을 나누어 격리성을 보장합니다. <br/>

### Q5. 격리 수준에 따라 발생하는 현상을 생각나는대로 설명해보세요.
격리 수준에 따라 발생하는 현상은 팬텀 리드, 반복 가능하지 않은 조회, 더티 리드가 있습니다.<br/>
- 팬텀 리드
  한 트랜잭션 내에서 동일한 쿼리를 보냈을 때 해당 조회 결과가 다른 경우를 말합니다. <br/>
  A가 어떤 결과를 조회하는 쿼리를 보냈는데 그 결과를 가져오기 전에 B가 그 결과값이 헤당되는 값을 하나 추가하게 된다면<br/>
  초기의 조회값이 A에게 가지 않고 B가 입력한 값 하나가 더 추가된 결과가 A에게 전달됩니다.<br/>
  결과적으로 A가 조회를 시작할 때와 조회 결과를 받을때 검색된 값이 다른 경우를 뜻합니다.<br/>
  
- 반복 가능하지 않은 조회
  반복 가능하지 않은 조회는 한 트랜잭션 내의 같은 행에 두 번 이상 조회가 발생했는데, 그 값이 다른 경우를 가리킵니다. <br/>
  조회를 반복했는데, 그 값이 다르니 반복이 가능하지 않은 거죠.<br/>
  예를 들어 사용자 1이 자신의 나이를 30으로 썼는데, 사용자 2가 사용자 1의 나이를 20으로 바꿔서 커밋해버리면 <br/>
  그 이후 사용자 1은 자신의 나이를 20으로 읽게 됩니다.<br/>
  
  팬텀 리드와 반복 가능하지 않은 조회가 다른 점은 값이 변화하느냐와 다른 값도 선택하느냐의 차이입니다.<br/>
  
- 더티 리드
  더티 리드는 반복 가능하지 않은 조회와 유사한데 <br/>
  한 트랜잭션이 실행 중일 때 다른 트랜잭션에 의해 수정되었지만 '아직 커밋되지 않은' 행의 데이터를 읽을 수 있을 떄를 말합니다. <br/>
  사용자 1이 자신의 나이를 30으로 썼는데, 사용자 2가 사용자 1의 나이를 20으로 바꿨지만 커밋하지 않았는데도  <br/>
  그 이후 사용자 1은 자신의 나이를 20으로 읽게 되는 경우를 말합니다.<br/>
  
### Q6. 격리 수준을 나누어 설명해보세요.
-SERIALIZABLE(순차적인)
  말 그대로 트랜잭션을 순차적으로 진행시킵니다. <br/>
  여러 트랜잭션이 동시에 같은 행에 접근할 수 없습니다.<br/>
  매우 엄격한 수준으로 해당 행을 격리 시키기 떄문에 교착 상태가 일어날 확률도 가장 높고 가장 성능이 떨어집니다.<br/>
  
-REPEATABLE_READ(반복가능한 조회)
하나의 트랜잭션이 수정한 행을 다른 트랜잭션이 수정할 수 없게 막지만 <br/>
새로운 행을 추가하는 것은 막지 않습니다. <br/>

-READ_COMMITTED(커밋된 트랜잭션 조회)
가장 많이 사용되는 격리 수준이며, 여러 데이터베이스에서 기본값으로 설저오디어 있습니다.<br/>
다른 트랜잭션이 커밋하지 않은 정보는 읽을 수 없습니다. <br/>
하지만 어떤 트랜잭션이 접근한 행을 다른 트랜잭션이 수정할 수 있어 반복 조회시 다른 내용을 발견할 수 있습니다. <br/>

-READ_UNCOMMITTED(커밋되지는 않았지만 변경된 트랜잭션 조회)
가장 낮은 격리 수준으로, 가장 빠릅니다. <br/>
데이터 무결성을 위해 되도록잉면 사용하지 않는 것이 좋지만, 거대한 양의 데이터를 <br/>
어림잡아 집계하는 데 사용하기 좋습니다. <br/>

### Q7. 무결성이 무엇인지 설명하고, 생각나는대로 종류를 말해보세요.
무결성이란 데이터의 정확성, 일솬성, 유효성을 유지하는 것을 말하며, 무결성이 유지되어야
데이터베이스에 저장된 데이터 값과 그 값에 해당하는 현실 세계의 실제 값이 일치하는지에 대한 신뢰가 생깁니다.
개체 무결성, 참조 무결성, 고유 무결성, NULL 무결성이 있습니다.

## 4. 데이터베이스의 종류 🍑

### Q1. 관계형 데이터베이스에 대해 간단히 설명하고, 종류를 말해보세요.
관계형 데이터베이스(RDBMS)는 행과 열을 가지는 표 형식 데이터를 저장하는 형태의 데이터베이스를 가리키며<br/>
SQL이라는 언어를 써써 조작합니다. MySQL, PostgreSQL, 오라클, SQL Server, MSSQL 등이 있습니다.<br/>

### Q2. NoSQL 데이터베이스에 대해 간단히 설명하고, 종류를 말해보세요
NoSQL(Not only SQL)이라는 슬로건에서 생겨난 데이터 베이스로 SQL을 사용하지 않는 데이터베이스를 말하며, <br/>
대표적으로 MongoDB와 redis 등이 있습니다.<br/>

### Q3. 사용한 데이터베이스가 있다면 어떤 경우에 사용했는지,왜 그 데이터베이스를 선택했는지,사용 경험을 얘기해주세요.
